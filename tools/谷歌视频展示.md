# Google 视频富结果实施指南

> **文档目标**：让文章页在 Google 搜索结果中显示为"视频结果"，而不仅仅是"文本结果带缩略图"

---

## 📋 快速检查清单

- [ ] VideoObject JSON-LD 已添加到页面 `<head>`
- [ ] 视频播放器已嵌入文章内容区
- [ ] 缩略图 URL 可访问（非 403/防盗链）
- [ ] 已通过 Rich Results Test 验证
- [ ] 已提交视频站点地图到 Search Console
- [ ] Search Console 视频索引报告显示"已索引"

---

## 🎯 实施目标与预期效果

### 当前问题
右侧只显示**小缩略图**（text result image），没有视频专属元素（时长条、上传日期、关键时刻）

### 目标效果
- ✅ Google 识别为"视频页面"
- ✅ 搜索结果显示视频缩略图 + 上传日期
- ✅ 可选：显示"关键时刻"时间戳

### 技术要求
1. 页面必须**可直接播放视频**（同一 URL）
2. 输出符合规范的 `VideoObject` 结构化数据
3. 视频时长 ≥ 30 秒
4. 允许 Google 抓取缩略图和视频文件

📖 [Google 视频结果文档](https://developers.google.com/search/docs/appearance/video)

## ✅ 必做步骤

### 任务 1：添加 VideoObject 结构化数据

**操作步骤**

在文章页面的 `<head>` 中添加 JSON-LD：

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "VideoObject",

  // 必填字段
  "name": "Chapter 6 Quest Walkthrough – 90s Quick Tips",
  "description": "Key steps to finish Chapter 6 fast.",
  "thumbnailUrl": [
    "https://i.ytimg.com/vi/YOUR_VIDEO_ID/maxresdefault.jpg"
  ],
  "uploadDate": "2025-11-24T08:00:00Z",
  "duration": "PT1M30S",  // ISO 8601 格式：1分30秒

  // 播放器 URL（YouTube 嵌入链接或自托管播放器）
  "embedUrl": "https://www.youtube.com/embed/YOUR_VIDEO_ID",

  // 推荐字段
  "contentUrl": "https://www.youtube.com/watch?v=YOUR_VIDEO_ID"  // 实际视频文件或页面 URL
}
</script>
```

**字段说明**

| 字段 | 要求 | 说明 |
|------|------|------|
| `name` | ✅ 必填 | 视频标题 |
| `description` | ✅ 必填 | 视频描述 |
| `thumbnailUrl` | ✅ 必填 | 缩略图数组，至少 160x90px |
| `uploadDate` | ✅ 必填 | ISO 8601 日期格式 |
| `duration` | ✅ 必填 | ISO 8601 时长格式（PT1M30S = 1分30秒）|
| `embedUrl` | ⭐ 推荐 | 嵌入播放器 URL |
| `contentUrl` | ⭐ 推荐 | 视频文件/页面 URL，有助于触发预览 |

**验证方法**

1. 访问 [Rich Results Test](https://search.google.com/test/rich-results)
2. 输入文章 URL
3. 确认识别到 **VideoObject** 类型
4. 检查无红色错误（黄色警告可接受）

---

### 任务 2：嵌入视频播放器

#### 方案 A：YouTube 嵌入（推荐）

**操作步骤**

在文章正文合适位置添加：

```html
<!-- 响应式 16:9 容器 -->
<div style="position: relative; padding-bottom: 56.25%; height: 0; overflow: hidden; border-radius: 12px; margin: 2rem 0;">
  <iframe
    src="https://www.youtube.com/embed/YOUR_VIDEO_ID?modestbranding=1&rel=0"
    title="Chapter 6 Quest Walkthrough – 90s Quick Tips"
    loading="lazy"
    allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share"
    allowfullscreen
    style="position: absolute; top: 0; left: 0; width: 100%; height: 100%; border: none;">
  </iframe>
</div>
```

**配置说明**
- `loading="lazy"`: 懒加载，减少首屏压力
- `modestbranding=1`: 减少 YouTube 品牌展示
- `rel=0`: 播放完不显示相关视频

**验证方法**
- 在浏览器中打开文章页，确认视频可正常播放

---

#### 方案 B：自托管视频（可选）

```html
<video
  controls
  preload="metadata"
  poster="https://your-domain.com/video-thumbnail.jpg"
  style="width: 100%; max-width: 800px; border-radius: 12px;">
  <source src="https://your-domain.com/video.mp4" type="video/mp4">
  <source src="https://your-domain.com/video.webm" type="video/webm">
  您的浏览器不支持 video 标签。
</video>
```

**注意事项**
- 需要提供视频文件的 `contentUrl`
- 自托管更容易触发"自动关键时刻"
- 注意带宽成本

### 任务 3：提交站点地图和监控

**操作步骤**

1. **创建视频站点地图**（可选但推荐）

在现有 sitemap.xml 之外，创建 `video-sitemap.xml`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"
        xmlns:video="http://www.google.com/schemas/sitemap-video/1.1">
  <url>
    <loc>https://www.totalchaos.info/article/chapter-6-guide</loc>
    <video:video>
      <video:thumbnail_loc>https://i.ytimg.com/vi/YOUR_VIDEO_ID/maxresdefault.jpg</video:thumbnail_loc>
      <video:title>Chapter 6 Quest Walkthrough – 90s Quick Tips</video:title>
      <video:description>Key steps to finish Chapter 6 fast.</video:description>
      <video:player_loc>https://www.youtube.com/embed/YOUR_VIDEO_ID</video:player_loc>
      <video:duration>90</video:duration>
      <video:publication_date>2025-11-24T08:00:00Z</video:publication_date>
    </video:video>
  </url>
</urlset>
```

2. **提交到 Search Console**
   - 进入 [Google Search Console](https://search.google.com/search-console)
   - 左侧菜单 → 站点地图
   - 添加 `video-sitemap.xml`

3. **监控索引状态**
   - 等待 1-3 天后检查
   - Search Console 左侧 → **视频索引报告**
   - 查看"已索引"数量和未索引原因

**验证方法**
- Search Console 视频索引报告显示相关 URL
- 状态为"有效"或"已索引"

## 🚀 可选增强：关键时刻（Key Moments）

### 什么是关键时刻？
搜索结果中显示视频的时间戳章节，用户可直接跳转到特定片段。

### 实现方式

#### 方式 1：手动标注片段（推荐）

在 `VideoObject` 中添加 `hasPart`：

```json
{
  "@context": "https://schema.org",
  "@type": "VideoObject",
  "name": "Chapter 6 Quest Walkthrough",
  // ...其他必填字段...

  "hasPart": [
    {
      "@type": "Clip",
      "name": "步骤一：进入任务区域",
      "startOffset": 0,
      "endOffset": 30,
      "url": "https://www.totalchaos.info/article/chapter-6-guide?t=0"
    },
    {
      "@type": "Clip",
      "name": "步骤二：击败守卫",
      "startOffset": 30,
      "endOffset": 60,
      "url": "https://www.totalchaos.info/article/chapter-6-guide?t=30"
    }
  ]
}
```

**前置条件**
- 文章页 URL 必须支持 `?t=秒` 参数
- 页面加载后自动跳转到对应时间戳

---

#### 方式 2：SeekToAction（自动识别）

```json
{
  "@context": "https://schema.org",
  "@type": "VideoObject",
  "name": "Chapter 6 Quest Walkthrough",
  // ...其他必填字段...

  "potentialAction": {
    "@type": "SeekToAction",
    "target": "https://www.totalchaos.info/article/chapter-6-guide?t={seek_to_second_number}",
    "startOffset-input": "required name=seek_to_second_number"
  }
}
```

**技术要求**
- URL 支持 `?t=` 跳转
- Google 能抓取到视频文件（自托管更佳）
- YouTube iframe 需使用 [IFrame Player API](https://developers.google.com/youtube/iframe_api_reference) 实现跳转

---

### 实现 URL 时间戳跳转（YouTube iframe 示例）

```javascript
// 监听 URL 参数并跳转视频时间
const urlParams = new URLSearchParams(window.location.search);
const startTime = parseInt(urlParams.get('t')) || 0;

// 使用 YouTube IFrame API
let player;
function onYouTubeIframeAPIReady() {
  player = new YT.Player('video-iframe-id', {
    events: {
      'onReady': (event) => {
        if (startTime > 0) {
          event.target.seekTo(startTime, true);
        }
      }
    }
  });
}
```

## ⚠️ 常见问题排查

### 问题 1：只显示小缩略图，没有视频样式

**原因**
- 未添加 `VideoObject` JSON-LD
- JSON-LD 格式错误或缺少必填字段

**解决方案**
- 用 Rich Results Test 检查
- 确认所有必填字段已填写

---

### 问题 2：缩略图不显示

**原因**
- `thumbnailUrl` 返回 403/404
- 图片被 `robots.txt` 屏蔽
- 图片尺寸小于 160x90px

**解决方案**
- 浏览器直接访问缩略图 URL，确认可访问
- 检查 `robots.txt` 是否允许抓取
- 使用高清缩略图（推荐 1280x720）

---

### 问题 3：关键时刻不生效

**原因**
- URL 不支持 `?t=` 跳转
- 未实现 JavaScript 跳转逻辑
- Google 未抓取到视频文件

**解决方案**
- 手动测试 `你的URL?t=30`，确认可跳转
- YouTube iframe 需使用 IFrame API
- 优先使用 `Clip` 方式（手动标注）

---

### 问题 4：Search Console 未出现视频索引报告

**原因**
- Google 未检测到任何视频页面
- 站点刚添加视频，需等待爬取

**解决方案**
- 使用 URL 检查工具手动提交页面
- 提交视频站点地图
- 等待 3-7 天
