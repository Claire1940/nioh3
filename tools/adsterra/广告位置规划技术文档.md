# Heartopia 网站广告位置规划技术文档

## 文档概述

本文档定义了 Heartopia 游戏网站所有页面的广告放置位置和实现方式,旨在平衡广告曝光和用户体验。

**核心原则:**
- 一个广告位在同一页面只能放置一次
- 每个页面至少放置 3 个广告
- 在尽可能多的不同页面放置广告,平衡曝光
- 严格遵循横幅(非原生)广告注意事项

---

## 可用广告类型

### 1. 原生横幅广告 (NativeBanner)
- **特点**: 自适应容器宽度,4:1 比例,无滚动条风险
- **实现方式**: JavaScript 脚本动态生成 DOM（**禁止使用 iframe**）
- **核心原理**: Adsterra 脚本自动检测容器宽度并动态生成适配的广告内容
- **关键注意**: **绝对不要限制高度**,让 Adsterra 脚本自由发挥
- **适用场景**: 页面顶部、Hero 下方、文章上方、任何需要响应式的位置

### 2. iframe 横幅广告 (AdBanner)
- **可用尺寸**:
  - `banner-300x250`: 中等矩形,适合内容中间
  - `banner-468x60`: 小型横幅,适合标题下方
  - `banner-728x90`: 大型横幅,适合页面顶部/底部
  - `banner-160x600`: 摩天大楼,适合侧边栏
- **实现方式**: iframe 嵌入独立 HTML 文件
- **关键注意**: 必须使用固定尺寸 + maxWidth: '100%',禁用滚动条

### 3. 社交栏广告
- **位置**: 插入在 `</body>` 标签上方
- **实现方式**: 全局脚本加载

## 实现注意事项

### 1. 原生横幅广告 (NativeBanner) 实现要点

**✅ 核心原理**:
- **禁止使用 iframe**: 原生横幅广告使用 JavaScript 脚本动态生成 DOM，不是 iframe 嵌入
- **Adsterra 脚本自动管理**: 脚本会自动检测容器宽度，动态生成适配尺寸的广告内容
- **直接 DOM 操作**: 广告内容直接插入到页面 DOM 中，不受 iframe 固定尺寸限制
- **智能尺寸计算**: Adsterra 会根据容器宽度计算合适的广告尺寸（通常保持 4:1 比例）

**✅ 必须遵守的规则**:
- **绝对不要限制高度**: 不要给原生横幅广告的容器设置固定高度、max-height 或 overflow: hidden
- **让 Adsterra 脚本自由发挥**: 脚本会自动检测容器宽度并生成适配的广告内容
- **提供足够空间**: 只需提供一个容器，Adsterra 会处理所有布局逻辑
- **无滚动条风险**: 因为内容是动态生成的，永远不会出现滚动条问题
- **0 风险显示一半**: Adsterra 脚本保证广告完整显示，不会被裁剪

**工作流程**:
```
1. React 渲染组件 → 提供容器 <div ref={containerRef}>
2. useEffect 执行 → 创建并插入 Adsterra 脚本
3. Adsterra 脚本加载 → 检测容器宽度（例如 800px）
4. 智能计算尺寸 → 生成适配的广告（例如 800×200px，保持 4:1）
5. 动态生成 DOM → 广告内容插入到容器中
6. 完美显示 → 自动适应容器，无需滚动，完美响应式
```

**正确示例**:
```tsx
// ✅ 正确：给足够空间，让脚本自由发挥
<div className="my-8">
  <NativeBanner />
</div>

// ✅ 正确：添加间距但不限制尺寸
<div className="container mx-auto px-4 my-8">
  <NativeBanner />
</div>
```

**错误示例**:
```tsx
// ❌ 错误：不要给固定高度
<div style={{ height: '100px' }}>
  <NativeBanner />
</div>

// ❌ 错误：不要用 overflow: hidden 裁剪
<div style={{ overflow: 'hidden', height: '150px' }}>
  <NativeBanner />
</div>

// ❌ 错误：不要限制最大高度
<div className="max-h-[200px]">
  <NativeBanner />
</div>
```

### 2. iframe 横幅广告 (AdBanner) 实现要点

**✅ 必须遵守的规则**:
- **使用固定尺寸**: iframe 必须使用固定的 width 和 height 属性
- **添加 maxWidth 限制**: 必须添加 `maxWidth: '100%'` 防止小屏幕溢出
- **禁用滚动条**: 必须添加 `scrolling="no"` 属性
- **居中对齐**: 使用 `flex justify-center` 或 `mx-auto` 居中显示

**正确示例**:
```tsx
// ✅ 正确：固定尺寸 + maxWidth
<div className="flex justify-center my-8">
  <AdBanner type="banner-728x90" />
</div>

// AdBanner 组件内部实现
<iframe
  src={`/ads/${type}.html`}
  width={config.width}
  height={config.height}
  style={{
    border: 'none',
    maxWidth: '100%',
    display: 'block'
  }}
  scrolling="no"
  title={`Adsterra ${type} Banner`}
/>
```

**错误示例**:
```tsx
// ❌ 错误：使用 padding-bottom 技巧
<div style={{
  position: 'relative',
  paddingBottom: `${(height / width) * 100}%`
}}>
  <iframe style={{ width: '100%', height: '100%' }} />
</div>

// ❌ 错误：没有 maxWidth 限制
<iframe width={728} height={90} style={{ border: 'none' }} />

// ❌ 错误：允许滚动条
<iframe width={728} height={90} scrolling="yes" />
```

### 3. 广告位唯一性原则

**核心规则**: 一个广告位在同一页面只能放置一次

**实现方式**:
- 每个广告位必须使用不同的 Adsterra key（从后台单独申请）
- 不要在同一页面复制粘贴相同的广告代码
- 每个页面的每个位置都应该有独立的广告位 ID

**示例**:
```tsx
// ✅ 正确：每个位置使用不同的广告位
<NativeBanner key="home-hero" />  // 使用 key 1
<AdBanner type="banner-728x90" key="home-middle" />  // 使用 key 2
<AdBanner type="banner-300x250" key="home-bottom" />  // 使用 key 3

// ❌ 错误：重复使用同一个广告位
<NativeBanner />
<NativeBanner />  // 同一个 key，违反规则
```

---

### 4. 响应式设计注意事项

**移动端处理**:
- 侧边栏广告（160x600）在移动端应该隐藏：`hidden lg:block`
- 大尺寸横幅（728x90）在移动端会自动缩放（因为有 maxWidth: '100%'）
- 原生横幅广告天然支持响应式，无需特殊处理

**示例**:
```tsx
// 侧边栏广告 - 移动端隐藏
<aside className="hidden lg:block lg:col-span-4">
  <AdBanner type="banner-160x600" />
</aside>

// 横幅广告 - 自动响应式
<div className="flex justify-center my-8">
  <AdBanner type="banner-728x90" />  {/* maxWidth: '100%' 自动处理 */}
</div>
```

## 最佳实践建议

### 1. 广告位置选择原则

**优先级排序**:
1. **首页 Hero 下方** - 流量最大，曝光率最高（使用原生横幅）
2. **文章详情页上方** - 用户深度阅读，停留时间长（使用原生横幅）
3. **文章详情页侧边栏** - 持续可见，适合摩天大楼广告
4. **分类导航页标题下** - 用户浏览入口，曝光率高
5. **页面底部** - 用户阅读完成后的自然位置

**避免的位置**:
- ❌ 不要在页面顶部导航栏附近放置广告（影响导航体验）
- ❌ 不要在表单输入区域附近放置广告（影响用户操作）
- ❌ 不要在重要 CTA 按钮附近放置广告（避免误点击）
- ❌ 不要在页面加载时立即显示过多广告（影响首屏体验）

---

### 2. 用户体验优化

**加载优化**:
- 使用懒加载技术，页面下方的广告延迟加载
- 首屏广告优先加载，非首屏广告延迟加载
- 添加预连接优化：`<link rel="preconnect" href="//www.highperformanceformat.com" />`

**视觉分隔**:
- 广告与内容之间保持足够的间距（建议 my-8 或 my-12）
- 使用边框或背景色区分广告区域（可选）
- 确保广告不会与页面内容混淆

**移动端优化**:
- 侧边栏广告在移动端隐藏
- 大尺寸横幅自动缩放适应屏幕
- 原生横幅广告天然支持移动端

---

### 3. 性能优化建议

**关键指标**:
- 首屏加载时间（LCP）< 2.5s
- 累积布局偏移（CLS）< 0.1
- 首次输入延迟（FID）< 100ms

**优化策略**:
1. **预留广告空间**: 为广告位预留固定高度，避免布局偏移
2. **异步加载**: 所有广告脚本使用 async 加载
3. **懒加载**: 非首屏广告使用 Intersection Observer 延迟加载
4. **预连接**: 在 layout.tsx 中添加 DNS 预解析和预连接

**示例代码**:
```tsx
// layout.tsx - 添加预连接
<head>
  <link rel="preconnect" href="//www.highperformanceformat.com" />
```

## 常见问题 FAQ

### Q1: 为什么原生横幅广告不能限制高度？为什么禁止使用 iframe？

**A**: 原生横幅广告的核心特点是使用 JavaScript 脚本动态生成 DOM，而不是 iframe 嵌入。

**工作原理**:
1. **Adsterra 脚本自动检测容器宽度**：脚本会检测外层容器的实际宽度
2. **智能计算广告尺寸**：根据容器宽度动态计算合适的广告尺寸（通常保持 4:1 比例）
3. **动态生成 DOM 元素**：直接在页面 DOM 中生成广告 HTML，不是 iframe
4. **自动适应容器**：内容可以自由伸缩，不受固定尺寸限制

**为什么不能限制高度**:
- 如果限制高度，Adsterra 生成的广告内容可能会被裁剪
- 脚本需要足够的空间来完整显示广告内容
- 限制高度会破坏 Adsterra 的智能布局算法

**为什么禁止使用 iframe**:
- 原生横幅广告本质上就是 JavaScript 脚本动态生成 DOM
- 使用 iframe 会变成固定尺寸的横幅广告，失去响应式特性
- iframe 需要手动设置尺寸，容易出现滚动条问题
- 原生横幅的优势就是自动适应，使用 iframe 会失去这个优势

**一句话总结**: 原生横幅广告由 Adsterra 脚本智能管理，会自动检测容器并生成适配的广告，所以不能限制高度，也禁止使用 iframe。你只需要提供一个容器，剩下的交给 Adsterra！

### Q2: 为什么 iframe 横幅广告不能使用 padding-bottom 响应式技巧？
**A**: Adsterra 的广告内容是固定尺寸的，不会自动缩放。使用 padding-bottom 技巧会导致容器高度计算错误，当容器宽度小于广告宽度时，会出现滚动条问题。正确做法是使用固定尺寸 + maxWidth: '100%'。

### Q3: 同一个广告位可以在多个页面使用吗？
**A**: 可以。同一个广告位可以在不同页面使用，但不能在同一个页面重复使用。例如，"首页 Hero 下方"的原生横幅广告位只能在首页使用一次，但"文章页标题上方"的原生横幅广告位可以在所有文章详情页使用。

### Q4: 移动端如何处理大尺寸广告？
**A**:
- 侧边栏广告（160x600）在移动端隐藏（使用 `hidden lg:block`）
- 大尺寸横幅（728x90）会自动缩放（因为有 maxWidth: '100%'）
- 原生横幅广告天然支持响应式，无需特殊处理

### Q5: 如何避免广告影响页面性能？
**A**:
1. 使用预连接优化：在 layout.tsx 中添加 preconnect 和 dns-prefetch
2. 非首屏广告使用懒加载（Intersection Observer）
3. 为广告位预留固定高度，避免布局偏移（CLS）
4. 所有广告脚本使用 async 加载

### Q6: 广告位置如何平衡曝光和用户体验？
**A**:
- 优先在高流量页面（首页、文章详情页）放置广告
- 避免在页面顶部导航栏、表单输入区域、重要 CTA 按钮附近放置广告
- 广告与内容之间保持足够的间距（my-8 或 my-12）
- 确保广告不遮挡核心内容的可读性
